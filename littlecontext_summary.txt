What We're Building

A Fail-Safe OTA Firmware Update System for ESP32. The core idea — when new firmware is pushed, the device downloads it, verifies it, boots into it in trial mode, runs a self-test, and only commits if everything works. If anything fails at any point — power cut, bad firmware, crash — the bootloader automatically rolls back to the previous working firmware. Device can never be permanently bricked.

What Gyanendra's Backend Does

MQTT Topic:   firmware/update
Broker:       localhost:1883
Message JSON:
  {
    "version":         "1.0.0",
    "file_name":       "firmware.bin",
    "file_size_bytes": 123456,
    "sha256_hash":     "abcdef...",
    "signature":       "base64...",
    "signing_alg":     "ed25519"
  }

Download URL: http://<server-ip>:8000/firmware/{version}/download

The firmware on ESP32 needs to subscribe to firmware/update, parse this JSON, download the binary via HTTP, verify SHA-256, flash to Partition B, reboot, self-test, then commit or let rollback happen.

```
## The 5-Step Flow Our Firmware Does
```
1. Boot → init NVS → connect WiFi
2. Subscribe to MQTT topic "firmware/update"
3. On message → parse JSON → download .bin from FastAPI server
4. Verify SHA-256 → write to Partition B → reboot
5. Post-reboot → self-test (WiFi + MQTT) →
        PASS → esp_ota_mark_app_valid_cancel_rollback() ✅
        FAIL → do nothing → bootloader auto-rollbacks ⏪
